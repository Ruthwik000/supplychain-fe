Build a production-grade frontend application called "AegisChain – Precision Logistics Intelligence".

This application is a real-time AI-powered logistics command center that orchestrates multiple intelligent agents (Packaging, Logistics, Route, Compliance, Audit). The UI must represent a continuous system lifecycle: input → validation → optimization → execution → monitoring → audit.

----------------------------------------
GLOBAL DESIGN SYSTEM
----------------------------------------

Theme:
- Dark, minimal, enterprise SaaS
- High contrast, low noise, grid-based layout

Colors:
- Background: #0A0A0A
- Surface/Card: #111827
- Primary: #1E3A8A
- Success: #22C55E
- Warning: #F59E0B
- Error: #EF4444
- Text Primary: #E5E7EB
- Text Secondary: #9CA3AF
- Borders: rgba(255,255,255,0.05)

Typography:
- Headings: Sora
- Body: Inter
- Labels: uppercase, letter-spacing 0.08em

Spacing:
- 8px grid system
- Card padding: 16–24px
- Section spacing: 32–64px

UI Components:
- Card (rounded 12px, subtle shadow)
- KPI tile
- Data table
- Timeline (audit)
- Progress bar (animated)
- Status badge (success/warning/error)
- Modal (for AI rationale)
- Stepper (multi-step flow)
- Map container (live ops)

Animations:
- Duration: 200–400ms
- Easing: ease-in-out
- Use for:
  - page transitions
  - hover elevation
  - progress updates
  - route transitions
- Avoid decorative animations

----------------------------------------
APPLICATION ARCHITECTURE
----------------------------------------

Routes:

/               → Landing Page
/dashboard      → System Overview
/shipments      → Shipment List
/shipments/new  → Multi-step Shipment Flow
/live-ops       → Real-time Operations
/audit          → Audit Intelligence Hub

Navigation Layout:
- Left sidebar:
  Overview
  Shipments
  Live Ops
  Audit Logs
- Top bar:
  Search input
  Notifications
  User profile

----------------------------------------
PAGE 1: LANDING
----------------------------------------

Sections:

Hero:
- Title: "Precision Logistics Intelligence"
- Subtitle describing AI agents + compliance
- CTA button: "Launch System"
- Background: animated grid or particle field

Features:
- 4 columns:
  Packaging Intelligence
  Logistics Optimization
  Adaptive Routing
  Compliance Guardrails

Metrics:
- KPI strip:
  latency, cost reduction, utilization, compliance rate

Footer:
- Minimal links

Behavior:
- CTA navigates to /dashboard

----------------------------------------
PAGE 2: DASHBOARD (/dashboard)
----------------------------------------

Purpose:
Provide system-wide visibility

Layout:

Row 1: KPI Cards
- Cost Savings
- Container Utilization
- Route Safety Score
- Active Shipments

Row 2:
- Line chart (logistics efficiency over time)
- Route risk breakdown (list with progress bars)

Row 3:
- Agent Telemetry Cards:
  Packaging AI (efficiency, latency)
  Logistics AI
  Route Engine
  Compliance AI

Row 4:
- Live Shipment Table:
  columns: ID, destination, status, risk, ETA

Primary Action:
- Button: "+ New Shipment" → /shipments/new

----------------------------------------
PAGE 3: SHIPMENT FLOW (/shipments/new)
----------------------------------------

Stepper Navigation:
Step 1 → Cargo Specifications
Step 2 → Vehicle Configuration
Step 3 → Route & Compliance
Step 4 → Final Review

----------------------------------------
STEP 1: CARGO SPECIFICATIONS
----------------------------------------

Inputs:
- Cargo Type (dropdown)
- Quantity
- Total Weight
- Total Volume
- Declared Value
- Hazardous Goods toggle

Right Panel:
- AI Insight box:
  dynamic text suggestions based on inputs
- Required Documents checklist

Actions:
- "Save Draft"
- "Continue to Logistics"

Behavior:
- On input change, simulate AI insight update

----------------------------------------
STEP 2: VEHICLE CONFIGURATION
----------------------------------------

Components:
- Vehicle category tabs (Heavy / Mid / Light)
- Efficiency score display
- Telemetry mini panel (uptime, sync)
- Visual grid placeholder for load layout

Right Panel:
- Manifest summary:
  cargo details
  estimated impact
  environmental metrics

Actions:
- "Initialize Route"

----------------------------------------
STEP 3: ROUTE & COMPLIANCE
----------------------------------------

Inputs:
- Origin node
- Destination node
- Transit priority (Standard / Express)

Route Options:
- List of routes with:
  distance
  duration
  efficiency score

Compliance Guardrails Panel:
- Hazmat Protocol
- Weight Limit Validation
- Insurance Check
- Restricted Zones

Each item:
- checkbox + status indicator

Behavior:
- If constraint fails → mark as error and disable route

Actions:
- "Preview Route Map"
- "Continue to Final Review"

----------------------------------------
STEP 4: FINAL REVIEW
----------------------------------------

Components:

Route Visualization:
- curved path line between origin and destination

Summary Panel:
- ETA
- Cost
- Confidence score
- Selected route name

Risk Indicators:
- Weather
- Congestion
- Maintenance

Comparison Table:
- Alternative routes with metrics

Actions:
- Checkbox confirmation
- Button: "Confirm & Dispatch"

----------------------------------------
AI EXECUTION STATE
----------------------------------------

After dispatch:

Display pipeline:
- Packaging Agent → processing
- Logistics Agent → optimizing
- Route Agent → validating
- Compliance Agent → checking

Each:
- status indicator + animation

Transition:
- on completion → redirect to /live-ops

----------------------------------------
PAGE 4: LIVE OPS (/live-ops)
----------------------------------------

Layout:

Main:
- Full-width map container
- shipment markers

Overlay Cards:
- Active shipments count
- In-transit value

Bottom Panel:
- Agent status cards:
  Packaging
  Logistics
  Route
  Compliance

Right Sidebar:
- Live audit feed (timeline)

Primary Action:
- "Raise Ticket"

Behavior:
- On click:
  simulate rerouting:
    old route fades
    new route animates
    notification appears

----------------------------------------
PAGE 5: AUDIT LOG (/audit)
----------------------------------------

Layout:

Top:
- Filters (time range, agent type, status)
- Export button

Main Table:
columns:
- timestamp
- agent
- action
- status
- actions

Row Interaction:
- "View Rationale" opens modal

Modal:
- detailed explanation text
- guardrails checked
- decision reason

Right Panel:
- Critical anomalies
- AI recommendations

----------------------------------------
INTERACTION LOGIC
----------------------------------------

State Flow:
Input → Validation → Optimization → Execution → Monitoring → Audit

Transitions:
- Step navigation: slide + fade
- Route update: path morph animation
- KPI updates: number count animation
- Audit logs: append with fade-in

----------------------------------------
FINAL REQUIREMENTS
----------------------------------------

- Maintain strict visual consistency
- Avoid template-like UI patterns
- Ensure strong spacing and alignment
- Keep interface minimal and data-focused
- Emphasize system intelligence through UI feedback
- Simulate real-time updates visually (no backend required)
